# 美妆「上妆试色」内容脚本生成

## 使用方式

用户提供一段美妆产品试色/上妆视频后，自动完成以下全部流程：

1. 下载视频
2. 提取音频脚本（miaoda-studio-cli speech-to-text）
3. 截取关键帧（ffmpeg fps=1/4 输出 JPG）
4. 按三段式框架输出完整分镜脚本
5. 关键帧与脚本节点对应
6. 打包 zip 交付

---

## 完整执行流程

### Step 1 — 下载视频

从飞书消息中提取 file_key，通过 feishu_im_bot_image 工具下载到本地：
- 保存路径：`/tmp/openclaw/bot-resource-{uuid}.mp4`
- 记录实际文件路径供后续使用

### Step 2 — 提取音频脚本

使用 miaoda-studio-cli 提取口播内容：

```bash
miaoda-studio-cli speech-to-text --file <音频路径> --lang zh --output json
```

音频可从视频中提取：
```bash
ffmpeg -i <视频路径> -vn -ac 1 -ar 16000 /tmp/video_audio.wav -y
```

### Step 3 — 分析视频结构

联合关键帧截图 + 音频脚本，识别以下结构：

- **开场钩子类型**：反差型 / 痛点型 / 期待型
- **核心叙事逻辑**：全程单眼画完（变装式）/ 分段对比式 / 逐步叠加式
- **功能展示顺序**：质地→演示→对比→持久 的实际排列
- **CTA 结构**：颜色推荐→套装→价格→行动指令 的实际顺序

### Step 4 — 截取关键帧

```bash
mkdir -p /tmp/video_frames_jpg
ffmpeg -i <视频路径> -vf "fps=1/4" -q:v 2 /tmp/video_frames_jpg/frame_%03d.jpg -y
```

如需更密集截图（每2秒一帧）：`fps=1/2`
如需特定节点截图：用 `-ss 时间` 精确定位

### Step 5 — 输出分镜脚本

按以下三个 PART 输出完整表格（不套代码块，直接发飞书消息）：

**PART 1 ｜ 开头（0 - Xs）**
目标：留人 + 建立期待

| 时间 | 场景/背景 | 机位/动作 | 口播内容 | 字幕要求 |
|------|-----------|-----------|---------|---------|
| ... | ... | ... | ... | ... |

**PART 2 ｜ 功能展示（Xs - Xs）**
目标：建立产品信任

- 2A：产品外观+质地
- 2B：上妆演示+刀锋刷修正
- 2C：效果对比+持久测试

**PART 3 ｜ 收尾CTA（Xs - Xs）**
目标：促成下单

### Step 6 — 关键帧与脚本对应

每个镜头节点后标注：
**→ 镜头参考：frame_XXX.jpg → ~Xs，画面描述**

### Step 7 — 打包交付

```bash
cd /tmp/video_frames_jpg
zip -r /tmp/openclaw/<产品名>_分镜脚本_关键帧.zip frame_*.jpg
```

通过 message 工具发送 zip 文件到飞书，消息正文附带分镜脚本全文。

## 三段式框架核心原则

```
开场（0-15%）→ 试色演示（60-75%）→ 收尾CTA（15-20%）
```

### 开场钩子类型

| 类型 | 模板句式 | 适用产品 |
|------|---------|---------|
| 素颜vs完妆反差 | 「不会画xx的老婆们看过来」 | 眼线、眉笔 |
| 效果前置 | 「用它3步搞定！」 | 任何易上手产品 |
| 痛点直击 | 「xx星人必看！」 | 护肤、美白 |
| 前后对比 | 「画vs不画，差的不是一点点」 | 眼线、眉笔、高光 |
| 工具导入 | 「xx想画好，工具一定要选对」 | 眼线、眉妆 |

### 功能展示模块（可组合）

1. **产品外观展示** → 特写/手持，口播类型+卖点
2. **质地测试** → 手背试色，口播触感+感受
3. **上脸演示** → 侧面近景+特写，动作词+位置描述
4. **效果对比** → 同框左右对比，大字字幕强化
5. **持久度测试** → 手指蹭/纸巾擦，场景+结果

### 收尾CTA四步

颜色推荐 → 套装内容 → 价格锚定 → 行动指令
